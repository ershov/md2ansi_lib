* ## Ticket: Task: Table cell wrapping when row exceeds line_width {#33}

      status: done
      created: 2026-05-21 05:03:09 UTC
      updated: 2026-05-21 05:36:33 UTC

  Reduce the table width to fit `state.line_width` (when > 0) by wrapping
  oversized cells.

  Layout algorithm:
    fixed   = columns whose natural width ≤ cell_min_width (never shrunk)
    fit_w   = line_width − overhead, where overhead = 3*N + 1
              (each cell adds 2 padding chars + 1 right rail; +1 left rail)
              fit_w -= sum(W[i] for i in fixed)
    wide    = [i not in fixed]
    if sum(W[wide]) <= fit_w: stop  # already fits
    loop:
      factor = fit_w / sum(W[wide])
      progressed = False
      for i in wide:
        new = int(W[i] * factor)
        if new <= cell_min_width:
          W[i] = cell_min_width; move i from wide -> fixed; progressed = True
        else:
          W[i] = new
      if progressed: re-enter loop (re-compute fit_w against the new fixed set)
      else: stop
    if total still overflows after all cells are pinned at cell_min_width:
      accept the overflow (per spec).

  Rendering (initial implementation):
  - Wrap **raw** cell text first with `_m2a_wrap_line(text, col_width, "")`,
    then re-render each wrapped sub-line through
    `_md2ansi(sub, current_style, M2A_CONTEXT_MD_INLINE, state)`.
    Add a code comment flagging the known caveat: any inline rule whose
    rendered width differs from raw text width (links `[x](url)` lose the
    URL; images `![alt](url)` become `[IMG: alt]`) will mis-account width.
    Tracked separately under [[fix-table-wrap-after-render]].
  - Pad each rendered sub-line to the column width via the alignment helper
    from [[table-column-alignment]] (left/right/center per column).
  - Row height = max(len(cell_lines[i])) over the row's cells.
    Top-align: shorter cells get trailing blank-line padding so all cells
    in a row finish at the same visual line.
  - Build the row by joining per-column sub-lines with `│`, one output line
    per sub-line.

  Options on `md2ansi(...)`:
  - `cell_min_width=20` — columns whose natural width is below this are
    never shrunk and never wrapped.
  - `row_dividers=None` — tristate:
      - `None` (default): auto. Emit `├─...─┼─...─┤` between body rows
        only if **any** body row produced multi-line cells.
      - `True`: always emit row dividers.
      - `False`: never emit row dividers.

  Non-goals / notes:
  - Cells are split on `|` before `_md2ansi` ever sees them, so inline
    rules can't bleed across cells without extra guarding.
  - If `line_width == 0`, skip the layout step entirely — render at natural
    widths (existing behavior).

  Tests:
  - Wide column shrinks; short column untouched.
  - Iterative pass: wide cell that drops below `cell_min_width` is pinned;
    remaining wide cells re-scaled.
  - Multi-line cell renders with top-aligned blank-line padding.
  - `row_dividers=True` forces dividers; `False` suppresses them.
  - `row_dividers=None` auto-enables dividers when any cell wraps; auto-omits
    them when no cell wraps.
  - All-cells-narrow table renders unchanged regardless of `line_width`.
  - Cell with `**bold**` stays styled across wrapped sub-lines.

      body: The initial table-cell-wrap (ticket #33) wraps **raw** cell text and then renders each wrapped sub-line through _md2ansi. This is wrong whenever an inline rule changes the visible width relative to the raw text:
